# 게임 룰북 JSON 구조 가이드

## 파일 목록
- `sabotage_rulebook.json`: 사보타지(Saboteur) 게임 룰북
- `rummikub_rulebook_gameonly.json`: 루미큐브 게임 룰북 (게임 규칙만)

## JSON 구조 설명

각 항목은 다음 필드를 포함합니다:

### 필수 필드
- **content**: 실제 룰북 내용 (RAG 검색 대상)
- **category**: 규칙 분류 (검색 최적화 및 필터링용)
- **rule_type**: 규칙 중요도
- **keywords**: 검색 키워드 배열 (RAG 검색 최적화용)

### category 예시 (사보타지) / 게임마다 달라질 수 있다.
- `게임 기본 정보`: 플레이어 수, 시간, 연령 등
- `팀 구성`: 역할 분배 방식
- `역할별 목표`: 각 팀의 목적
- `승리 조건`: 승리 방법
- `게임 플레이`: 진행 방식
- `정보 제한`: 숨겨진 정보 규칙
- `최종 승리 조건`: 게임 전체 승리

### rule_type 분류
- `"필수"`: 꼭 알아야 하는 핵심 규칙
- `"기본"`: 일반적인 게임 진행 규칙
- `"참고"`: 예시나 부가 설명

## 작성 가이드라인

1. **content 작성**
   - 1-3문장으로 간결하게 작성
   - 하나의 개념/규칙당 하나의 항목
   - 독립적으로 이해 가능한 단위로 분할

2. **keywords 작성**
   - 자주 검색될 만한 용어들로 구성
   - 동의어, 관련어 포함
   - 5-10개 내외 권장

3. **category 명명**
   - 명확하고 직관적인 이름 사용
   - 게임별 특성에 맞게 조정
   - 일관성 있는 분류 체계 유지
